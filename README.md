## 📌 Описание цикла

Цикл состоит из 4 последовательных практик, охватывающих полный жизненный цикл создания микросервисного приложения — от архитектурного проектирования до развёртывания в Kubernetes с системой мониторинга. На всех этапах используются ИИ-ассистенты (GigaChat, YandexGPT, Copilot и др.).

## 🧭 Краткое содержание практик

### Практика №9 – Архитектурное проектирование с ИИ в нотации C4
- Составление Problem Statement
- Генерация диаграмм C1 (контекст), C2 (контейнеры), C3 (компоненты) через ИИ
- Ручная доработка и анализ «галлюцинаций» ИИ
- Результат: `context.puml`, `container.puml`, `component.puml` + PNG

### Практика №10 – Реализация MVP с использованием ИИ
- Разработка двух микросервисов (API Gateway + Worker)
- Интеграция с PostgreSQL и Redis
- Автотесты (pytest)
- Docker Compose для локального запуска
- Результат: работающий код, тесты, `docker-compose.yml`

### Практика №11 – Контейнеризация и деплой в Kubernetes
- Сборка Docker-образов
- Написание YAML-манифестов (Deployment, Service, Ingress, ConfigMap, Secret)
- Развёртывание в Minikube
- Результат: запущенный кластер, доступ через Ingress

### Практика №12 – Мониторинг и наблюдаемость
- Установка Prometheus + Grafana (kube-prometheus-stack)
- Экспорт метрик из приложения (HTTP-запросы, latency, бизнес-метрики)
- Создание дашборда
- Результат: работающий мониторинг, скриншоты дашбордов
